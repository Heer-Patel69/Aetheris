# ADR-006: Recovery And State Persistence Are Mandatory Runtime Boundaries

Status: Accepted
Owner: Production Readiness Board
Approval Date: 2026-07-01

## Context
Autonomous execution can fail during code generation, review, or external operations.

## Problem
Without checkpointing and recovery the platform loses work and cannot resume safely.

## Decision
Persist checkpoints, classify failures, generate patch plans, and route git operations through explicit engine contracts.

## Alternatives
- Best-effort retry only
- Manual operator recovery

## Trade-Offs
- More artifact writes
- Crash-safe execution

## Consequences
- Interrupted runs must produce resume metadata
- Unsafe patching must fail closed
