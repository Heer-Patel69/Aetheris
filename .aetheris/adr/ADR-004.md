# ADR-004: Planning Engines Remain Monolithic Until Contract Decomposition Is Funded

Status: Accepted
Owner: Chief Software Architect
Approval Date: 2026-07-01

## Context
SPEC-009 through SPEC-031 are currently implemented in a shared planning module.

## Problem
Immediate physical decomposition would risk unrelated behavior churn.

## Decision
Document the monolith as an interim implementation boundary and require contract-safe extraction later.

## Alternatives
- Split immediately
- Leave undocumented

## Trade-Offs
- Preserves current tests
- Creates known module-size debt

## Consequences
- Future extraction must preserve public APIs and artifact contracts

## Evidence
- `src/intelligence/planners.py`
- `tests/test_planners.py`
