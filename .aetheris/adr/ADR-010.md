# ADR-010: Security Is Fail-Closed Across Documentation And Runtime Contracts

Status: Accepted
Owner: Principal Security Engineer
Approval Date: 2026-07-01

## Context
Aetheris reads untrusted repositories and can generate code.

## Problem
Secrets leakage, path traversal, prompt injection, and fabricated claims are systemic risks.

## Decision
Treat project content as untrusted evidence, redact secrets, enforce workspace boundaries, and fail closed on policy violations.

## Alternatives
- Warn-only security
- Trust local workspace

## Trade-Offs
- May block ambiguous requests
- Prevents unsafe release artifacts

## Consequences
- Security controls must appear in docs, tests, and runbooks
