# ADR-001: Aetheris Operates As An Autonomous Software Engineering Operating System

Status: Accepted
Owner: Architecture Review Board
Approval Date: 2026-07-01

## Context
Aetheris is governed as an ASE-OS rather than a prompt wrapper.

## Problem
Without an operating-system boundary the repo would collapse into isolated scripts and documentation fragments.

## Decision
Preserve the kernel, knowledge, planning, execution, and intelligence layers as first-class platform subsystems.

## Alternatives
- Treat Aetheris as a CLI utility
- Treat Aetheris as a generic agent library

## Trade-Offs
- Higher governance overhead
- Clearer ownership and stronger production posture

## Consequences
- All major subsystems require RFC/SPEC ownership
- Quality gates become mandatory for production readiness
