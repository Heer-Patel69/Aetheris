# ADR-005: Execution Uses DAG Scheduling And Bounded Parallelism

Status: Accepted
Owner: Principal Software Engineer
Approval Date: 2026-07-01

## Context
Autonomous execution requires safe task ordering.

## Problem
Linear execution wastes concurrency and unsafe parallel execution corrupts files.

## Decision
Use decomposed tasks, dependency graphs, topological ordering, critical paths, and file-lock-aware parallel batches.

## Alternatives
- Linear queue
- Unbounded parallel workers

## Trade-Offs
- More planning overhead
- Safer execution and better recovery

## Consequences
- Cycle detection blocks invalid plans
- Parallel batches must declare locked files
