# RFC-003: Autonomous Engineering Engine (AEE)

STATUS: Frozen
Priority: Critical
Version: 1.0

======================================================================
1. MISSION & PURPOSE
======================================================================
The Autonomous Engineering Engine (AEE) executes the authoritative planning blueprint generated by RFC-002. AEE acts as a coordinator, delegating code generation, patch application, and compilation recovery loops to specialized skill sets and LLM backends without manual user orchestrations.

======================================================================
2. CORE ARCHITECTURE
======================================================================
The execution engine runs through the following pipeline:

### Wave 1: Execution Primitives
1. **SPEC-032: Task Decomposition Engine (TDE)**: Parses the blueprint's task queue and constructs modules, features, stories, and task matrices.
2. **SPEC-033: Dependency Graph Builder (DGB)**: Constructs topological Directed Acyclic Graphs (DAG), detects cycle errors using DFS coloring, and calculates critical paths.
3. **SPEC-034: Skill Selection Engine (SSE)**: Matches decomposed tasks to the most qualified specialist skills.

### Wave 2: Execution Intelligence
4. **SPEC-035: Model Routing Engine (MRE)**: Selects the optimal LLM (e.g. Claude 3.5 Sonnet, Gemini 1.5 Pro) based on cost, latency, and quality.
5. **SPEC-036: Context Assembly Engine (CAE)**: Compiles minimal, complete context payloads under token budget limits.
6. **SPEC-037: Execution Scheduler (ES)**: Determines task execution times managing priorities and dynamic rescheduling on failures.

### Wave 3: Autonomous Engineering
7. **SPEC-038: Parallel Execution Engine (PEE)**: Groups independent tasks into parallel execution batches using file lock planners.
8. **SPEC-039: Autonomous Code Generation Engine (ACGE)**: Safely edits and creates source files, logging reasons to the EKB journal.
9. **SPEC-040: Self Review Engine (SRE)**: Evaluates file modifications against styling guidelines and performance targets.

### Wave 4: Recovery & Continuity
10. **SPEC-041: Patch & Recovery Engine (PRE)**: Corrects code compiler warnings and test failures with AST-safe patches.
11. **SPEC-042: State Persistence Engine (SPE)**: Serializes checkpointers and snapshots to memory/disk.
12. **SPEC-043: Git Operations Engine (GOE)**: Manages version control commits formatting task metrics and blueprint tags.

======================================================================
3. DATA CONTRACTS & INTERFACES
======================================================================

#### Input Interface:
- Consumes `execution.input.json` containing the tasks and their dependency mappings:
```json
{
  "tasks": [
    {
      "id": "task_db_init",
      "dependencies": []
    },
    {
      "id": "task_auth_pipeline",
      "dependencies": ["task_db_init"]
    }
  ]
}
```

#### Output Interfaces:
- Emits execution progression steps and status reports into EKB history logs.
- Outputs `execution_state.json` documenting task progressions.

======================================================================

======================================================================

- **Mock Execution**: Simulate compilation failure alerts to verify rollback recovery routines.
