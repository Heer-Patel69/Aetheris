# Aetheris Platform — User Stories

## Personas
1. **Principal Architect**: Needs to review high-level RFC structures, trace specifications down to individual skills, and examine the full dependency graph.
2. **Specialist Developer**: Needs to read skill specifications, inspect inputs and outputs, download templates, and view related code examples.
3. **Core Operator**: Needs to monitor live kernel runs, debug configuration states, and inspect the decision logs.

## Stories
* **US-1 (Search everything)**: As an Architect, I want to search for "sre" or "validation" globally and instantly find related skills, specs, and docs, so that I don't waste time hunting in folders.
* **US-2 (Interactive Relationships)**: As a Developer, when I am reading a SPEC, I want to click on mentioned RFCs or other SPECs and have the interface navigate immediately, so that I can trace the design context.
* **US-3 (Downloads Package)**: As an Operator, I want to download any specialist skill in YAML or a full ZIP package (MD + JSON + YAML) in one click, so that I can install it locally.
* **US-4 (Dynamic Updates)**: As a Developer, when I add a new specification file `rfcs/SPEC-171.md`, I want the dev server to automatically reload and display the new specification in the sidebar, so that I see my changes immediately.
* **US-5 (Apple Storytelling)**: As a Guest, I want to scroll through a beautifully animated storytelling landing page to understand the product DNA of Aetheris, so that I get wowed by its quality.
